# 小区党支部"一点办"便民服务品牌 - 软件系统设计方案

## 一、项目概述

### 1.1 项目背景
基于《关于打造小区党支部"一点办"便民服务品牌的实施方案》，设计一套线上线下融合的社区治理数字化平台。

### 1.2 核心理念
**支部引领、科技赋能、一点办成、服务到家**

### 1.3 总体架构
**"一核双线三融合"**
- 一核：小区党支部领导核心
- 双线：线上智慧平台 + 线下服务矩阵
- 三融合：线上线下深度融合，全流程闭环管理

### 2.2 技术栈选型

### 3.2 核心功能模块

#### 3.2.1 诉求办理模块（一点·报诉求）

**功能描述**：居民一键上报问题，平台自动派单，实现"接诉即办"

**核心功能**：
| 功能点 | 说明 |
|--------|------|
| 一键报事 | 拍照上传、语音输入、定位自动获取 |
| 智能分类 | AI自动识别诉求类型（物业/治安/环境/设施等） |
| 自动派单 | 根据类型、位置自动派给对应责任人 |
| 进度跟踪 | 实时查看处理进度，关键节点推送通知 |
| 满意度评价 | 5星评价+文字反馈，纳入考核 |
| 催办提醒 | 超时自动催办，升级预警 |

**业务流程**：
```
居民上报 → 智能分类 → 自动派单 → 责任人接单 → 现场处理 → 居民确认 → 评价反馈
              ↓           ↓            ↓           ↓           ↓
           AI识别      规则引擎      短信/推送    拍照留痕    满意度评分
```

#### 3.2.2 便民服务模块（一点·约服务）

**功能描述**：整合优质服务商，提供一站式便民服务

**核心功能**：
| 功能点 | 说明 |
|--------|------|
| 红色便民仓 | 开锁、维修、家政等服务商入驻，党支部审核准入 |
| 明码标价 | 服务价格公示，杜绝乱收费 |
| 一键预约 | 选择服务、时间，在线预约 |
| 上门服务 | 服务商接单后上门，全程可追溯 |
| 政务自助 | 对接区行政审批局，社保/医保/养老等高频事项 |
| 微心愿 | 特殊群体发布需求，党员志愿者认领 |

**服务商管理**：
- 入驻申请 → 党支部审核 → 签订协议 → 培训上岗 → 服务监督 → 积分考核

#### 3.2.3 邻里互动模块（一点·邻里圈）

**功能描述**：打造线上社区，促进邻里交流

**核心功能**：
| 功能点 | 说明 |
|--------|------|
| 红色议事厅 | 线上发起投票、公示结果（公共收益、加装电梯等） |
| 邻里互助 | 闲置交换、拼单团购、宠物互助等 |
| 活动报名 | 社区活动发布、在线报名、签到打卡 |
| 邻里圈 | 社区朋友圈，分享生活、求助问答 |
| 公告通知 | 物业通知、党建动态、政策宣传 |

#### 3.2.4 积分激励模块（一点·评实效）

**功能描述**：双向积分激励，提升参与积极性

**积分规则**：
| 角色 | 积分场景 | 积分值 |
|------|---------|--------|
| **居民** | 上报诉求 | +5分/次 |
| | 参与投票 | +2分/次 |
| | 邻里互助 | +10分/次 |
| | 服务评价 | +1分/次 |
| | 活动参与 | +3分/次 |
| **党员** | 志愿服务 | +10分/小时 |
| | 帮办代办 | +15分/次 |
| | 议事协商 | +5分/次 |
| | 微心愿认领 | +20分/次 |

**积分使用**：
- 线上商城兑换实物
- 线下合作商户抵扣现金（超市、理发店、洗衣店、洗车店）

#### 3.2.5 数据可视化模块

**功能描述**：服务热力图、数据分析、决策支持

**核心功能**：
| 功能点 | 说明 |
|--------|------|
| 服务热力图 | 按季度生成，展示诉求办结率、满意度排名 |
| 诉求分析 | 类型分布、趋势分析、热点问题 |
| 党员考核 | 服务积分、居民评议、排名公示 |
| 书记直通车 | 居民直接向社区党委书记反映问题 |
| 实时监控 | 待办诉求、超时预警、满意度指标 |

### 4.2 核心表结构

#### 用户表 (t_user)
```sql
CREATE TABLE t_user (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    phone VARCHAR(20) UNIQUE NOT NULL COMMENT '手机号',
    password VARCHAR(100) COMMENT '密码',
    nickname VARCHAR(50) COMMENT '昵称',
    avatar VARCHAR(255) COMMENT '头像',
    real_name VARCHAR(50) COMMENT '真实姓名',
    id_card VARCHAR(18) COMMENT '身份证号',
    community_id BIGINT COMMENT '所属小区ID',
    building VARCHAR(20) COMMENT '楼栋',
    unit VARCHAR(10) COMMENT '单元',
    room VARCHAR(20) COMMENT '房号',
    role ENUM('resident','party_member','grid_worker','property','admin') DEFAULT 'resident',
    status TINYINT DEFAULT 1 COMMENT '状态:0禁用1启用',
    points INT DEFAULT 0 COMMENT '积分余额',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

#### 诉求表 (t_appeal)
```sql
CREATE TABLE t_appeal (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    appeal_no VARCHAR(32) UNIQUE COMMENT '诉求编号',
    user_id BIGINT NOT NULL COMMENT '诉求人ID',
    type VARCHAR(50) COMMENT '诉求类型:物业/治安/环境/设施/其他',
    title VARCHAR(200) COMMENT '标题',
    content TEXT COMMENT '诉求内容',
    images JSON COMMENT '图片数组',
    location VARCHAR(255) COMMENT '位置描述',
    lat DECIMAL(10,8) COMMENT '纬度',
    lng DECIMAL(11,8) COMMENT '经度',
    status TINYINT DEFAULT 0 COMMENT '状态:0待处理1处理中2待确认3已完成4已关闭',
    priority TINYINT DEFAULT 1 COMMENT '优先级:0低1中2高3紧急',
    handler_id BIGINT COMMENT '处理人ID',
    handler_type ENUM('grid_worker','property','party_member','volunteer','department') COMMENT '处理人类型',
    assign_time DATETIME COMMENT '派单时间',
    handle_time DATETIME COMMENT '开始处理时间',
    finish_time DATETIME COMMENT '完成时间',
    confirm_time DATETIME COMMENT '居民确认时间',
    result TEXT COMMENT '处理结果',
    result_images JSON COMMENT '处理结果图片',
    score TINYINT COMMENT '满意度评分1-5',
    comment TEXT COMMENT '评价内容',
    is_overtime TINYINT DEFAULT 0 COMMENT '是否超时:0否1是',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```
